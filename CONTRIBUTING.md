# Contributing to Edutive Open Learning Toolkit

Thank you for your interest in contributing to Edutive Open Learning Toolkit.

This project is still in an early stage, so contributions to documentation, examples, schemas, and project direction are very welcome.

## Ways to contribute

You can contribute by:

- improving documentation,
- adding sample questions,
- suggesting better dataset fields,
- improving the JSON schema,
- building validation tools,
- writing tests,
- creating examples,
- improving the roadmap,
- or reporting bugs and confusing documentation.

## Good first contributions

If you are new to the project, you can start with:

- fixing typos,
- improving README sections,
- adding 3 to 5 sample questions,
- adding a new subject example,
- improving `docs/dataset-format.md`,
- or suggesting validation rules.

## Dataset contribution rules

When contributing sample questions, please make sure that:

1. the question is written by you or allowed to be shared,
2. the correct answer is clearly marked,
3. the explanation is included,
4. the topic and difficulty are provided,
5. and the question follows the current dataset format.

## Development workflow

Recommended workflow:

1. Fork this repository.
2. Create a new branch.
3. Make your changes.
4. Test or review your changes.
5. Open a pull request.

Example branch names:

```text
docs/improve-dataset-format
feature/question-validator
example/add-math-questions
fix/readme-typo
```

## Pull request guidelines

A good pull request should include:

- a clear title,
- a short explanation of the changes,
- screenshots if the change affects UI,
- examples if the change affects data format,
- and related issue links if available.

## Commit message style

Recommended commit examples:

```text
docs: improve dataset format explanation
feat: add basic question validator
fix: correct sample answer key
test: add validation tests for missing fields
example: add linear function questions
```

## Project values

This project values:

- clear documentation,
- beginner-friendly contribution,
- ethical use of educational data,
- respect for student privacy,
- practical tools over unnecessary complexity,
- and open collaboration.

## Questions

If something is unclear, please open an issue and describe what you are trying to do.
